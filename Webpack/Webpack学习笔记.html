webpack可以在终端中使用，其最基础的命令是  

  1. //若全局webpack
  2. webpack app/main.js public/bundle.js

  1. //webpack非全局安装的情况
  2. node_modules/.bin/webpack app/main.js public/bundle.js

通过配置文件来使用webpack


  1. module.exports = {
  2. 	entry:  __dirname + "/app/main.js",	//已多次提及的唯一入口文件
  3. 	output: {
  4. 		path: __dirname + "/public",	//打包后的文件存放的地方
  5. 		filename: "bundle.js"			//打包后输出文件的文件名
  6. 	}
  7. }

当前文件夹打开命令行    webpack

如果文件夹名字不一样或者多个配置文件可以使用--config指定配置文件
  1. webpack --config webpack.config.js



通过配置package.json文件来引导任务执行

 
npm的start是个特殊的脚本名称可以直接通过npm start执行，其他名字的时候必须npm run (scripts name) 例如npm run dev



Source Maps调试

 
开发阶段用eval-source-map构建速度更快

生产阶段用source-map  生成单独的且功能完成的文件，具有最好的Source Map    但是会减慢打包文件构建速度
